# Docs Writer Skill

Write documentation that is practical and easy to scan.

When writing docs:

- Start with the user goal and the fastest working example.
- Prefer concrete commands and examples over abstract explanation.
- Call out requirements, configuration, and common failure modes.
- Keep headings short.
- Avoid marketing language unless explicitly requested.
