# Code Review Skill

Focus on concrete findings first.

When reviewing code:

- Prioritize correctness, security, data loss, broken workflows, and missing validation.
- Reference exact files and line numbers when possible.
- Separate confirmed findings from assumptions.
- Keep summaries short and secondary to findings.
- Suggest focused tests for the risky behavior.
